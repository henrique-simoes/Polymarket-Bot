1.  Minimum order size: Orders must meet a minimum size threshold (you'll get `INVALID_ORDER_MIN_SIZE` error if too small). The `min_order_size` is returned in the `get-book` endpoint response.
    
2.  Market orders work differently: To place a "market order" via the API, you submit a limit order with a marketable price (one that crosses the spread). The API uses FOK (Fill-Or-Kill) or FAK (Fill-And-Kill) order types for immediate execution.
    
3.  Order delays: Some markets (like sports and likely 15-minute crypto markets) have a `seconds_delay` parameter that delays marketable orders.
    
4.  Taker fees: 15-minute crypto markets have taker fees enabled (up to 1.56% at 50% probability), which must be included in your signed order via `feeRateBps`.
    

The website may handle these requirements automatically, while the API requires you to explicitly meet minimum size requirements and include proper fee parameters.

Key points:

-   `signature_type`: Use `0` for EOA, `1` for Magic Link, `2` for Gnosis Safe/browser wallet
-   `funder`: Your proxy wallet address (find at polymarket.com/settings)
