Based on the documentation, I can now answer your question about the CLOB API behavior for 15-minute crypto markets.

---

Yes, the CLOB API has different behavior than the website for 15-minute crypto markets:

1.  Minimum order size: Orders must meet a minimum size threshold (you'll get `INVALID_ORDER_MIN_SIZE` error if too small). The `min_order_size` is returned in the `get-book` endpoint response.
    
2.  Market orders work differently: To place a "market order" via the API, you submit a limit order with a marketable price (one that crosses the spread). The API uses FOK (Fill-Or-Kill) or FAK (Fill-And-Kill) order types for immediate execution.
    
3.  Order delays: Some markets (like sports and likely 15-minute crypto markets) have a `seconds_delay` parameter that delays marketable orders.
    
4.  Taker fees: 15-minute crypto markets have taker fees enabled (up to 1.56% at 50% probability), which must be included in your signed order via `feeRateBps`.
    

The website may handle these requirements automatically, while the API requires you to explicitly meet minimum size requirements and include proper fee parameters.

To check if a 15-minute market has settled and which outcome won:

### 1\. CLOB API - `getMarket()` method

```
from py_clob_client.client import ClobClientclient = ClobClient(host="https://clob.polymarket.com", chain_id=137)market = client.get_market(condition_id)# Check if resolvedif market["closed"]:    for token in market["tokens"]:        if token["winner"]:            print(f"Winner: {token['outcome']}")  # "Yes" or "No"            print(f"Winning token ID: {token['token_id']}")
```

### 2\. WebSocket - `market_resolved` message

Subscribe to the market channel with `custom_feature_enabled` flag to receive real-time resolution notifications:

```
{  "event_type": "market_resolved",  "winning_asset_id": "...",  "winning_outcome": "Yes"}
```

### Key fields in the Market response:

-   `closed`: `true` when market is resolved
-   `tokens[].winner`: `true` for the winning outcome token

The API doesn't provide the specific strike price the 15-minute market needed to beat—that's typically in the market description via the Gamma API.

The error is about decimal precision for your order amounts. Here's the fix:

---

The Problem:

-   `makerAmount` (USDC) supports max 2 decimals
-   `takerAmount` (shares) supports max 4 decimals

Your order has `5.66 @ 0.53` which likely produces amounts with too many decimals.

The Fix - Round your amounts:

```
import mathdef round_order_amounts(price, size):    # Round price to tick size (e.g., 0.01)    price = round(price, 2)        # For BUY: makerAmount = price * size (USDC, 2 decimals)    maker_amount = math.floor(price * size * 100) / 100  # 2 decimals        # takerAmount = size (shares, 4 decimals)      taker_amount = math.floor(size * 10000) / 10000  # 4 decimals        return price, taker_amount# Exampleprice, size = round_order_amounts(0.53, 5.66)
```

Also note: The Python client uses `create_and_post_order()` (not `create_and_post_market_order()`). For market orders, use:

```
# Create market order with FOKsigned_order = client.create_market_order(    UserMarketOrder(        token_id=token_id,        amount=3.00,  # Dollar amount for BUY        side=BUY,        fee_rate_bps=0    ))resp = client.post_order(signed_order, OrderType.FOK)
```
